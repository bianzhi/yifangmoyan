# 墨岩K线 — 产品设计文档

> 产品定位：面向A股投资者的桌面端K线分析工具，核心价值是将缠论、威科夫两大经典理论**可视化**到K线图上，用算法自动标注关键信号，辅助投资者识别买卖时机。

---

## 1. 产品目标

| 维度 | 目标 |
|------|------|
| **核心价值** | 缠论笔/线段/中枢/三类买卖点 + 威科夫吸筹派发关键信号，**一键标注，直观可读** |
| **目标用户** | 熟悉缠论或威科夫理论的A股投资者（非入门级用户） |
| **使用场景** | 日常盯盘选股：打开某只股票 → 切换周期 → 查看缠论/威科夫信号 → 辅助决策 |
| **差异化** | Mac原生体验 + 两大理论融合 + 本地数据不泄露 + 开源免费 |

### 2.2 缠论分析（核心卖点一）

**可视化要素定义：**

| 缠论概念 | 图形表达 | 交互 | 优先级 |
|----------|----------|------|--------|
| **分型** | 顶分型▲绿色 / 底分型▼黄色 小三角（可独立显示/隐藏） | 无特殊交互 | P1 |
| **笔** | 顶底分型之间的折线段，上升笔红色、下降笔蓝色，线宽2px | 笔悬停显示：方向、起止价、幅度、包含K线数 | P0 |
| **线段** | 比笔更粗的折线（3px），上升段红色虚线、下降段蓝色虚线 | 线段悬停：方向、幅度；**点击线段→弹出次级别走势** | P0 |
| **笔中枢** | 矩形区域，半透明灰色填充(0.15) + 实线边框 | 悬停显示：zg/zd/gg/dd、中枢级别、进入/离开段数量 | P0 |
| **线段中枢** | 矩形区域，半透明橙色填充(0.15) + 虚线边框 | 同笔中枢 | P1 |
| **背驰标记** | 在背驰点标注⚡图标 + "背驰"文字 | 悬停显示：背驰类型（笔/线段背驰）、前后力度对比数据 | P0 |
| **一买** | 绿色▲圆标 + "1B" 文字 | 悬停：买点依据（哪段背驰触发） | P0 |
| **二买** | 浅绿▲圆标 + "2B" 文字 | 悬停：买点依据 | P0 |
| **三买** | 深绿▲圆标 + "3B" 文字 | 悬停：买点依据（哪个中枢上方回踩） | P0 |
| **一卖** | 红色▼圆标 + "1S" 文字 | 悬停：卖点依据 | P0 |
| **二卖** | 浅红▼圆标 + "2S" 文字 | 悬停：卖点依据 | P0 |
| **三卖** | 深红▼圆标 + "3S" 文字 | 悬停：卖点依据 | P0 |

**勾选面板（左侧或右侧浮动面板）：**

```
┌──── 缠论分析 ────┐
│ ☑ 分型            │
│ ☑ 笔              │
│ ☑ 线段            │
│ ☑ 笔中枢          │
│ ☐ 线段中枢        │
│ ☑ 买卖点          │
│ ☑ 背驰            │
│ ── 高级 ──        │
│ ☐ 区间套分析      │  ← Phase 3
│ ☐ 走势递归        │  ← Phase 3
└───────────────────┘
```

**关键交互：点击线段→次级别走势**

1. 用户在日线上点击某一线段
2. 右侧面板展开该线段对应时间段的**60F K线 + 60F笔/中枢**
3. 自动滚动到线段起始位置
4. 面包屑导航：`日线 > 线段#3 > 60F`

## 5. 交互设计

### 5.1 核心操作流

```
选择股票 → 自动加载日线K线 + 缠论/威科夫标注
    ↓
切换周期 → 平滑过渡，保留缩放位置
    ↓
悬停笔/线段/中枢/信号 → 弹出信息卡
    ↓
点击线段 → 右侧展开次级别走势
    ↓
勾选/取消覆盖层 → 即时刷新
```

### 5.2 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| `1`-`8` | 切换周期：月/周/日/60F/30F/15F/5F/1F |
| `/` 或 `Cmd+F` | 聚焦搜索框 |
| `←` / `→` | 向左/右平移K线 |
| `+` / `-` | 放大/缩小 |
| `Cmd+S` | 添加到自选股 |
| `B` | 只显示缠论笔 |
| `D` | 只显示缠论线段 |
| `W` | 切换威科夫标注 |

### 5.3 缠论/威科夫切换模式

提供三种预设视图，一键切换：

| 模式 | 说明 |
|------|------|
| **纯K线** | 不叠加任何标注，纯净看盘 |
| **缠论模式** | 显示笔/线段/中枢/买卖点/背驰 |
| **威科夫模式** | 显示TR/冰线/关键信号/阶段 |
| **融合模式** | 两大理论全部叠加 + 融合解读 |

## 6. 算法要求

### 6.1 缠论算法必须严格实现

| 算法模块 | 当前状态 | 目标要求 |
|----------|----------|----------|
| 去包含关系 | ✅ 基础实现 | 对齐 czsc 0.9.9 remove_include，处理边界情况 |
| 分型识别 | ✅ 基础实现 | 支持顶分型/底分型，严格检查高低点关系 |
| 笔的构建 | ⚠️ 简化实现 | **必须对齐 czsc：至少5根合并K线、笔必须顶底交替、处理不完备笔** |
| 线段构建 | ⚠️ 简化实现 | 特征序列法 + 破坏规则，对齐缠论原文定义 |
| 笔中枢 | ⚠️ zg/zd计算有误 | 修正：zg=min(三段高点), zd=max(三段低点)，中枢扩展需实现 |
| 线段中枢 | ⚠️ 同上 | 同笔中枢 |
| 背驰检测 | ⚠️ 仅MACD面积 | 需增加：价格幅度背驰、区间套、趋势背驰vs盘整背驰 |
| 买卖点 | ⚠️ 仅一买一卖 | **三类买卖点必须完整实现，严格对齐缠论定义** |
| 走势分解 | ❌ 未实现 | 递归分解：笔→线段→1F走势→5F走势→30F走势→日线走势 |

### 6.2 威科夫算法需增强

| 算法模块 | 当前状态 | 目标要求 |
|----------|----------|----------|
| 阶段识别 | ⚠️ 简单均值 | 需结合P&F（点数图）或结构分析 |
| 事件识别 | ⚠️ 阈值过粗 | 需改进上下文关联（SC→AR→ST 必须有因果链） |
| 交易区间 | ⚠️ 固定窗口 | 需支持变长区间、自动合并相邻区间 |
| 趋势线 | ⚠️ 只连2点 | 需支持3点验证、通道线、延长线 |

## 7. 非功能需求

| 维度 | 要求 |
|------|------|
| **性能** | 1000根K线 + 缠论全分析 < 200ms；图表渲染60fps |
| **数据量** | 支持5500+股票 × 8级别，本地Parquet存储 |
| **可靠性** | 数据源降级自动切换，单源失败不影响使用 |
| **隐私** | 全部数据本地存储，不上传任何用户数据 |
| **平台** | macOS 13+（优先），后续可扩展Windows/Linux |
| **体积** | 安装包 < 30MB（Tauri WebView 方案） |
| **更新** | 支持自动更新（Tauri updater） |
