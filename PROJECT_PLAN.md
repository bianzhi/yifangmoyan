## Phase 2：缠论核心 (预估 3 周)

### 目标
缠论算法严格对齐 czsc 0.9.9，三类买卖点完整实现，前端可清晰看到笔/线段/中枢/买卖点/背驰。

### 2.1 缠论算法修正 (后端，第1-2周)

| # | 任务 | 优先级 | 当前问题 | 交付物 |
|---|------|--------|----------|--------|
| 2.1.1 | 修正去包含关系 | P0 | 低点比较逻辑在下降趋势取max有误 | 对齐czsc remove_include，增加边界case单测 |
| 2.1.2 | 修正分型识别 | P0 | 基本正确，需增加"分型经过包含处理后再确认"逻辑 | check_fxs 增加 stricter 校验 |
| 2.1.3 | 重写笔的构建 | P0 | 缺少最少5根K线限制、不完备笔处理、笔延伸逻辑 | 对齐czsc check_bi，支持新笔/老笔切换 |
| 2.1.4 | 重写线段构建 | P0 | 特征序列法实现不完整 | 对齐czsc，实现特征序列+顶底分型破坏判别 |
| 2.1.5 | 修正中枢识别 | P0 | zg/zd取值有误；缺少中枢扩展逻辑 | zg=min(三段高点), zd=max(三段低点)；支持9段扩展中枢 |
| 2.1.6 | 增强背驰检测 | P0 | 仅MACD面积，缺价格幅度/区间套 | 支持：MACD面积背驰 + 价格幅度背驰 + 区间套初步 |
| 2.1.7 | 重写买卖点识别 | P0 | 仅一买一卖，三类买卖点不完整 | 严格一买/二买/三买/一卖/二卖/三卖 |
| 2.1.8 | 增加线段级别分析 | P1 | 仅在笔级别做背驰和买点 | 线段中枢 + 线段背驰 + 线段级别买卖点 |

**验收标准：**
- 选取10只典型股票（含趋势股、震荡股、次新股），对比 czsc Python 版本结果
- 笔/线段/中枢匹配度 > 90%
- 买卖点位置偏差 < 3根K线
- 核心算法单测覆盖率 > 80%

### 2.2 缠论前端覆盖层 (前端，第2-3周)

| # | 任务 | 说明 |
|---|------|------|
| 2.2.1 | 笔线段渲染 |折线段绘制，上升红/下降蓝，支持悬停高亮 |
| 2.2.2 | 中枢矩形渲染 | 半透明矩形 + 边框，支持zg/zd标注 |
| 2.2.3 | 买卖点标记 | 圆形图标 + 文字标签（1B/2B/3B/1S/2S/3S） |

### 目标
威科夫关键信号准确识别，前端可清晰看到TR/冰线/Spring/JOC/LPS/UTAD等标注。

### 3.1 威科夫算法增强 (后端，第1周)

| # | 任务 | 说明 |
|---|------|------|
| 3.1.1 | 重写阶段识别 | 结合结构高点和量价特征，替代简单均值法；支持吸筹A/B/C/D子阶段 |
| 3.1.2 | 重写事件识别 | 事件须有因果链（SC→AR→ST→Spring），不能独立出现；增加上下文窗口 |

| 3.1.4 | 增强趋势线 | 3点验证趋势线 + 通道线 + 延长到当前K线 |
| 3.1.5 | 事件关联图谱 | 记录事件之间的因果/时序关系，供前端展示 |

- 对比3只经典吸筹股（如2023年中远海控），SC/AR/ST/Spring位置人工确认
- 对比3只派发股（如2023年宁德时代顶部），UTAD/SOW位置人工确认

## 风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 缠论算法与czsc结果差异大 | 中 | 高 | 逐函数对比测试，选取典型case回归 |
| 威科夫事件识别误报率高 | 高 | 中 | 采用保守策略，宁漏报不误报；可调参数 |
| lightweight-charts覆盖层性能差 | 中 | 高 | 备选方案：Canvas自绘 或 切换ECharts |
| Tushare API限流 | 中 | 低 | 已有多源降级；增加请求间隔控制 |
| 前端交互复杂度超预期 | 中 | 中 | 分阶段交付，先做核心交互，高级交互延后 |

## 下一步行动（Phase 2 立即开始）

1. **重写笔的构建** — 这是缠论的基础，当前实现缺少最少5根K线限制和笔延伸逻辑
2. **修正中枢计算** — zg/zd取反问题，影响所有基于中枢的判断
3. **重写买卖点识别** — 三类买卖点必须完整
4. **编写对比测试** — 用czsc Python版本的结果做golden test

> 原则：**先准确，再高效，最后好看**。算法不对，界面再美也是空中楼阁。
