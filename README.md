# 墨岩K线 — A股K线分析系统

> 类同花顺的 macOS 桌面端 A 股 K 线分析应用，集成 **缠论** + **威科夫量价分析** + **融合解读**

## ✨ 功能特性

### 📊 专业 K 线图表

- 基于 TradingView **lightweight-charts**，支持缩放、拖拽、十字线
- 8 级时间周期：月 / 周 / 日 / 60F / 30F / 15F / 5F / 1F
- 阳线红、阴线绿，适配 A 股习惯
- 主图 + 成交量副图 + MACD 副图，十字线联动
- 性能优化：>5000 根 K 线自动截断渲染，保持 60fps 流畅

### 🔴 缠论分析（严格对齐缠论原著 & czsc 0.9.9）

| 模块 | 说明 |
|------|------|
| 去包含关系 | 对齐 czsc `remove_include`，处理边界情况 |
| 分型识别 | 顶分型 ▲ / 底分型 ▼，严格高低点关系检查 |
| 笔的构建 | 至少 5 根合并 K 线、顶底交替、处理不完备笔 |
| 线段分析 | 特征序列法 + 破坏规则，对齐缠论原文定义 |
| 中枢识别 | zg/zd 修正（zg=min 三段高点，zd=max 三段低点），中枢扩展 |
| 背驰检测 | 趋势背驰 + 盘整背驰 + 价格幅度背驰 |
| 三类买卖点 | 一买一卖 / 二买二卖 / 三买三卖，笔级+线段级 |
| 走势递归 | 笔→线段→1F 走势→5F 走势→30F 走势→日线走势 |
| 区间套 | 多级别联立信号 |

**可视化覆盖层：** 分型三角、笔折线（红升蓝降）、线段虚线（3px）、笔/线段中枢矩形、买卖点圆标（1B/2B/3B/1S/2S/3S）、背驰⚡标记

### 🟣 威科夫分析（严格对齐威科夫三大法则）

| 法则 | 实现模块 | 说明 |
|------|----------|------|
| **供需法则** | supply_demand.rs | 供给线 + 需求线 + 突破判断 |
| **因果法则** | trading_range.rs | 事件驱动交易区间 + 自动合并 |
| **努力与结果** | effort.rs | 量价背离检测 + 异常量分析 |

**11 种事件识别：**

| 事件 | 标签 | 阶段 |
|------|------|------|
| PSY 初步供给 | 🟡 PSY | 派发早期 |
| BC 买入高潮 | 🟡 BC | 派发 |
| AR 自动反弹 | 🟢 AR | 吸筹早期 |
| ST 二次测试 | 🔵 ST | 吸筹确认 |
| SC 卖出高潮 | 🔴 SC | 吸筹启动 |
| Spring 弹簧 | ⭐ Spring | 吸筹确认 |
| SOS 强势信号 | 💪 SOS | 吸筹→拉升 |
| LPS 最后支撑 | 🛡 LPS | 拉升回调 |
| JOC 跳过小溪 | 🚀 JOC | 拉升确认 |
| SOW 弱势信号 | ⚠️ SOW | 派发→下跌 |
| UTAD 派发冲高 | ⚠️ UTAD | 派发结束 |
| LPSY 最后供给 | 🔴 LPSY | 派发晚期 |

**5 子阶段自动识别：** 吸筹A/B/C → 拉升 → 派发A/B/C → 下跌

### ⚡ 缠论+威科夫融合解读（核心创新）

当缠论买卖点与威科夫事件在 **±5 根 K 线窗口** 内同时出现时，自动生成融合信号：

| 缠论信号 | 威科夫信号 | 融合解读 | 信号强度 |
|----------|------------|----------|----------|
| 一买 | Spring | 底部反转双重确认 | ★★★★★ |
| 一买 | SC | 背驰+卖出高潮见底 | ★★★★ |
| 二买 | LPS | 回调确认吸筹完成 | ★★★★ |
| 三买 | JOC | 突破确认趋势延续 | ★★★★ |
| 一卖 | UTAD | 顶背离+派发冲高 | ★★★★★ |
| 二卖 | SOW | 反弹无力+弱势信号 | ★★★★ |
| 三卖 | LPSY | 中枢破坏+结构破位 | ★★★★ |
| 买点 | AR | 底部反转确认 | ★★★ |
| 卖点 | PSY | 顶部反转预警 | ★★★ |

无精确规则匹配时，生成通用融合信号（★2），确保不遗漏任何关联。

### 🔄 智能数据同步

- **四大数据源自动协同**：东方财富 → 新浪 → 腾讯 → Tushare，主源失败秒切备源
- **启动自动后台同步**：应用启动即增量更新所有本地股票，失败项自动重试最多 5 轮直到 0 失败
- **非阻塞同步**：后端线程执行同步，前端 1 秒轮询进度，不阻塞图表浏览和搜索
- **RwLock 并发优化**：读操作（图表/搜索）用 `.read()`，写操作用 `.write()`，分析计算不持锁
- **按板块同步**：上证主板 / 深证主板 / 创业板 / 科创板，一键批量同步
- **数据校验**：OHLC 逻辑校验 + 缺口检测 + 跨数据源交叉验证

### 🏷️ 自选股与搜索

- **拼音搜索**：输入 `zgpa` → 中国平安，300ms 防抖实时联想

## 🔄 数据源

四大 A 股数据源自动协同，主源失败自动切换备源：

| 数据源 | 级别支持 | 特点 | 最大条数 |
|--------|----------|------|----------|
| 东方财富 | 1分~日线 | 分钟级最优 | 10,000 |
| 新浪财经 | 1分~日线 | 速度快 | 2,000 |
| 腾讯财经 | 1分~周线 | 稳定 | 2,000 |
| Tushare | 1分~月线 | 数据质量最高 | — |

**同步策略**：
- 分钟级：东方财富 → 新浪 → 腾讯 → Tushare
- 日线/周线/月线：新浪 → 腾讯 → Tushare → 东方财富
- 周线/月线：优先直接获取，失败则从日线重采样确保完整性

**自动同步**：
- 应用启动时自动增量同步所有本地股票
- 失败项自动重试最多 5 轮，直到 0 失败
- 后台线程执行，不阻塞 UI
- 顶部按钮显示同步状态（绿色脉冲点表示进行中）

## 📊 数据校验

系统提供三层数据校验：

1. **OHLC 逻辑校验** — 检查开高低收的逻辑一致性（High ≥ Open/Close/Low 等）
2. **缺口检测** — 识别交易日历中的异常缺口
3. **跨数据源交叉校验** — 新浪与腾讯数据对比，发现差异超过 0.5% 的记录

校验结果按 0~1 评分，1.0 为完美数据。

## 🏛️ 架构亮点

### 非阻塞同步设计

```
前端                          后端线程
  │                              │
  │── start_sync_board() ──────→ │ 启动同步
  │   (立即返回)                  │
  │                              │ ├── 同步股票1
  │← get_sync_status() ──────── │ │ (更新进度)
  │   (1秒轮询)                  │ ├── 同步股票2
  │                              │ │ ...
  │← get_sync_status() ──────── │ ├── 自动重试失败项
  │   (retry_round: 1)          │ │ (最多5轮)
  │                              │ └── 完成
  │← get_sync_status() ──────── │ running: false
  │   (同步完成)
```

### RwLock 并发优化

| 操作 | 锁类型 | 持锁时间 |
|------|--------|----------|
| `get_chart_data` 读取 K 线 | `.read()` | ~1ms（复制数据后释放） |
| 缠论/威科夫/融合分析 | 无锁 | 数十~数百ms |
| `search_stocks` | `.read()` | ~1ms |
| `sync_stock` 写入文件 | 不持锁 | N/A（独立路径） |
| `set_data_dir` | `.write()` | ~1ms |
