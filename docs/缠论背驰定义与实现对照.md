> 对照实现: `crates/czsc/src/beichi.rs` + `analyzer.rs`

## 一、缠论原文对"背驰"的定义

### 1. 核心定理 (第15课 / 24课)

**"没有趋势，没有背驰"** — 背驰只存在于趋势中，盘整中只有"盘整背驰"（另一个概念）。

> **缠中说禅背驰-买卖点定理**：任一背驰都必然制造某级别的买卖点，任一级别的买卖点都必然源自某级别走势的背驰。

两层含义：
- 有背驰 → 必有逆转（回到前中枢或反向走势）
- 有买卖点 → 必源自某级别背驰（第一类由趋势背驰构成，第二/三类由盘整背驰构成）

### 2. 趋势背驰的标准结构 (第37课精确定义)

```
a + A + b + B + c
```

**必须同时满足 5 个条件：**

| # | 条件 | 原文出处 |
|---|------|----------|
| ① | a+A+b+B+c 是趋势，A、B 同级别中枢，无重叠、同方向递进 | 37课："A、B是同级别的中枢，否则只能看成较大中枢的震荡" |
| ② | 至少两个中枢，背驰发生在第二个中枢之后 | 27课："肯定不会发生在第一个中枢之后，至少第二个中枢之后" |
| ③ | c必须包含对B的第三类买卖点（否则只是B的次级别震荡→盘整背驰） | 37课："c必然是次级别的，至少包含对B的一个第三类买卖点" |
| ④ | 上涨:c必创新高 / 下跌:c必创新低 | 37课："c一定要创出新高/新低，否则用盘整背驰处理" |
| ⑤ | c段力度 < b段力度（力度 = MACD柱子面积） | 15课/24课："后一个趋势力度比前一个弱" |

**力度比较的精确对象**（37课明确）：
比较的是 **最后两个中枢之间的连接段 b_last** 与 **c 段**，不是 A 中枢前的 a 段。原文："在 a+A1+b1+A2+b2+...+An+c 中，力度比较的是 b(n-1) 与 c。"

### 3. 力度的衡量方法 (第15课 / 24课)

缠论给了两种力度衡量方式：

**方法一（标准）：趋势力度**
> "前一'吻'的结束与后一'吻'开始由短线均线与长期均线相交所形成的面积"
> 缺点：必须等再次接吻后才能判断，滞后

**方法二（辅助）：MACD 柱子面积**
> "用MACD判断背驰，首先要有两段同向的趋势"
> MACD 柱子面积比较：C段红柱面积 < A段红柱面积 → 顶背驰

**标准背驰形态**（27课原文）：
> "回抽0轴的黄白线再次下跌不创新低，而且柱子的面积是明显小于第1段的，
>  一般来说，只要其中一个符合就可以是一个背弛的信号，两个都满足就更标准了。"

即 **两个独立判断标准**（满足其一即可）：
- 标准A：MACD柱子面积缩小
- 标准B：黄白线回抽0轴后不创新高/低

### 4. 盘整背驰（独立概念，不是真正的"背驰"）

结构：`a + A + b`（单中枢）

- a = 中枢前进入段，b = 中枢后离开段
- 条件：b力度 < a力度，a、b同方向
- 意义：中枢震荡力度衰减
- 两种情况（24课）：
  - 破位盘整背驰：b段破a段极值 → 可能形成第三类买卖点
  - 未破位盘整背驰：b段未破a段极值 → 更可能回到中枢

> "盘整背驰与背驰是两回事，不能混为一谈。"（53课）

### 5. 多级别联立（区间套）（第27课）

> "某大级别的转折点，可以通过不同级别背驰段的逐级收缩范围而确定。"
> 先找大级别背驰段，再在次级别找背驰段，逐级精确定位。

## 二、当前实现对照

### 2.1 整体架构

```
analyzer.rs:
  K线 → 去包含 → 分型 → 笔 → 线段 → 中枢 → 背驰 → 买卖点
                                    ↓         ↓
                              bi_zs + xd_zs  detect_bi_beichi()
                                             detect_xd_beichi()
```

背驰只在 **笔级别** 和 **线段级别** 检测，两个级别完全独立。

### 2.2 核心逻辑 (`detect_beichi_inner`）

1. 将段列表转为 `TrendSection`（方向、起止、价格极值）
2. 计算每个段的 MACD 力度（= 柱子面积）
3. 按中枢递进方向分组 `group_zs_by_trend`
4. 组内 ≥2 个中枢 → 趋势背驰 `check_trend_backdivergence`
5. 组内 =1 个中枢 → 盘整背驰 `check_panzheng_backdivergence`

### 2.3 逐条件对照

| 原文条件 | 实现状态 | 代码位置 |
|----------|----------|----------|
| ① 至少2个同级别同方向无重叠中枢 | ✅ 已实现 | `group_zs_by_trend` + 递进检查 |
| ② 背驰在第二个中枢之后 | ✅ 已实现 | 只对 ≥2 中枢的 group 做趋势背驰 |
| ③ c 包含对 B 的第三类买卖点 | ✅ 已实现 | `cond_third_bs`: c段最低点 > B.zd / c段最高点 < B.zg |
| ④ c 创新高/新低 | ✅ 已实现 | `cond_new_extreme` |
| ⑤ c段力度 < b段力度 (面积) | ✅ 已实现 | `cond_area` |
| 盘整背驰: b力度 < a力度 | ✅ 已实现 | `check_panzheng_backdivergence` |
| 破位/未破位区分 | ✅ 已实现 | `broken` flag |

### 2.4 力度计算 (`calculate_section_power`）

```rust
// 上涨: 只累加 MACD histogram 正值 (红柱)
area = sum(macd_hist[start..=end].filter(|v| v > 0))

// 下跌: 只累加 |负值| (绿柱取绝对值)
area = sum(macd_hist[start..=end].filter(|v| v < 0).map(|v| v.abs()))
```

与原文的"MACD柱子面积"一致。

## 三、发现的问题

### ❌ 问题 1：缺少"黄白线回抽0轴"的标准B判断

**原文**（27课）：
> "回抽0轴的黄白线再次下跌不创新低，而且柱子的面积是明显小于第1段的，
>  一般来说，只要其中一个符合就可以是一个背弛的信号"

当前实现 **只看柱子面积**，完全不检查 DIF/DEA（黄白线）是否回抽0轴。这导致大量"面积缩小但不构成标准背驰"的假信号被漏掉，也可能漏掉"黄白线不创新高/低但面积不缩小"的真正背驰。

### ❌ 问题 2：只有两个级别，缺少走势级别背驰

缠论的背驰是 **走势级别** 的概念，不是笔/线段级别的概念。原文的背驰示例都是在"30分钟趋势"、"日线趋势"等时间级别上讨论的。

当前实现：
- `bi_beichi`：笔级别 — 笔是构造元素，不是标准走势级别
- `xd_beichi`：线段级别 — 线段 ≈ 1F走势，但只在日K/30F K线上检测

**日K上的笔** 构造出来的背驰信号，在缠论原文框架里没有明确定义。原文强调：
- 1F走势背驰 → 1F级别买卖点
- 5F走势背驰 → 5F级别买卖点
- 30F走势背驰 → 30F级别买卖点

当前缺少走势递归后的背驰检测（即从线段递归出1F走势→5F走势后在走势上检测背驰）。

### ❌ 问题 3：缺少趋势背驰的"盘整背驰兜底"

37课原文：
> "如果a+A+b+B+c是上涨，c一定要创出新高；...否则，就算c包含B的第三类买卖点，
>  也可以对围绕B的次级别震荡用盘整背驰的方式进行判断。"

当前代码：c没创新高/新低 → `return None`（什么也不输出）。
应该：c没创新高/新低时，回退到盘整背驰来检测，而不是直接放弃。

同样，c不包含第三类买卖点时也是直接 `return None`，应该回退到盘整背驰。

### ❌ 问题 4：缺少区间套多级别确认

27课区间套定理：大级别背驰必须由次级别背驰确认。

当前实现：笔和线段级别的背驰独立检测，没有做"线段背驰需要笔背驰确认"的区间套验证。这导致低级别噪音产生大量假信号。

### ⚠️ 问题 5：中枢分组方向判定依赖相邻中枢对

`group_zs_by_trend` 用"后中枢 zd > 前中枢 zg"判定上涨递进。这在标准趋势中枢中是正确的，但中枢可能有扩张/扩展的情况，此时相邻中枢之间可能有重叠但方向未变。

### ⚠️ 问题 6：b 段定位可能不准确

趋势背驰比较的是"最后两个中枢之间的连接段 b"和"c"。当前代码在 sections 中查找 `start_idx >= second_last_zs.end_index && start_idx < last_zs.start_index` 的段。但 b 段可能有多个段（b 本身可以是趋势），应该取最后一个离开前中枢的段，或者取所有同向段的综合力度。

## 四、改进建议

| 优先级 | 改动 | 影响 |
|--------|------|------|
| 🔴 P0 | 增加黄白线回抽0轴判断（标准B），与面积判断为 OR 关系 | 大幅减少假信号 |
| 🔴 P0 | 趋势背驰条件不满足时回退到盘整背驰（而非 return None） | 恢复被漏掉的信号 |
| 🟡 P1 | 在走势递归后增加走势级别背驰检测（不再只依赖笔/线段级别） | 这才是原作定义的背驰 |
| 🟡 P1 | 增加区间套：线段背驰 = 线段趋势满足 + 对应笔级别趋势也满足 | 多级别确认 |
| 🟢 P2 | b段定位：支持多段b的力度合并 | 边缘case修正 |
| 🟢 P2 | 中枢扩展/扩张的方向判定 | 边缘case修正 |
