# 缠论+威科夫融合信号解读

## 什么是融合信号？

**融合信号**是指：当**缠论买卖点**与**威科夫事件**出现在同一位置（±5 根 K 线窗口内）时，系统自动将两者关联，生成一条"双体系确认"的高置信度信号。

简单来说：两个独立的理论体系在同一个时刻发出相同方向的信号 → 互相印证 → 信号可信度更高。

---

## 黄色星标的含义

图表上的黄色 ⭐ 标记就是融合信号。星星数量代表信号强度：

| 星数 | 含义 | 匹配方式 |
|------|------|----------|
| ★★★★★ (5星) | 最强确认 | 精确规则匹配（如 一买+Spring、一卖+UTAD） |
| ★★★★ (4星) | 强确认 | 精确规则匹配（如 二买+LPS、三买+JOC） |
| ★★★ (3星) | 中等确认 | 精确规则匹配（阶段转换类） |
| ★★ (2星) | 弱确认 | 通用匹配（缠论买卖点附近有威科夫事件但无精确规则） |

**颜色含义**：
- 🟡 **金黄色 `#ffd700`** → 看多信号（`bullish`），标记在 K 线**下方**
- 🟠 **橙色 `#ff6d00`** → 看空信号（`bearish`），标记在 K 线**上方**

## 通用匹配（2星）

当缠论买卖点附近有威科夫事件，但不在上述精确规则表中时，自动生成**通用融合信号**（强度 2 星）：

> "缠论X买/X卖附近出现威科夫事件XX"

## 技术实现

### 数据流

```
缠论分析 (CzscAnalyzer)  →  CzscResult { buy_sell: Vec<BuySellPoint> }
威科夫分析 (WyckoffAnalyzer)  →  WyckoffResult { events: Vec<WyckoffEvent> }
                                    ↓
                           融合引擎 (fusion::analyze_fusion)
                                    ↓
                            FusionResult { signals: Vec<FusionSignal> }
```

### 匹配逻辑（`src-tauri/src/fusion.rs`）

1. 遍历每个缠论买卖点
2. 在 ±5 根 K 线窗口内查找威科夫事件
3. 按规则表从上到下匹配，命中第一条后停止（同一买卖点不重复匹配）
4. 如果精确规则不匹配但有附近事件 → 生成 2 星通用融合

### 前端渲染（`src-ui/src/App.vue` → `renderFusionOverlays`）

- 用 `★` 字符重复 `strength` 次（如 3 星 → `★★★`）
- 配合文字 `融合` 组成图标注解（如 `★★★★ 融合`）
- 叠加在 K 线图的 candle series markers 上
