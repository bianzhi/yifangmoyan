# 三类买卖点审查文档

对照缠论原文（17/20/21/27/29/33/43课），逐条审查 `buy_sell.rs` 的实现逻辑。

---

## 第一部分：缠论原文对三类买卖点的定义

### 一、第一类买卖点

**17课（走势终完美）原文：**

> "缠中说禅技术分析基本原理一"：任何级别的任何走势类型终要完成。
> "走势终完美"——走势类型完成以后，就会转化为其他类型的走势。

> 由定理一，可以得到第三类买卖点定理：一个次级别走势类型向上离开缠中说禅走势中枢，然后以一个次级别走势类型回试，其低点不跌破ZG，则构成第三类买点...

17课首先定义了走势中枢、盘整、趋势，并给出了走势终完美的原理。但17课本身没有明确定义第一类买卖点。一买一卖的定义散见于后续课程。

**27课（盘整背驰与历史性底部）原文：**

> "趋势，一定有至少两个同级别中枢，对于背驰来说，肯定不会发生在第一个中枢之后，肯定是至少是第二个中枢之后"
> "如果在第一个中枢就出现背驰，那不会是真正意义上的背驰，只能算是盘整背驰"
> "第一类买点，多数由趋势的背驰构成"

**29课（转折的力度与级别）原文：**

> "缠中说禅背驰-转折定理：某级别趋势的背驰将导致该趋势最后一个中枢的级别扩展、该级别更大级别的盘整或该级别以上级别的反趋势。"

**43课（有关背驰的补习课）原文：**

> "某级别的背驰必然导致该级别原走势类型的终止，进而开始该级别或以上级别的另外一个走势类型。"

**归纳——第一类买卖点的严格定义：**

| | 定义 |
|--|------|
| **一买** | 下跌趋势（≥2个同级别中枢）出现趋势底背驰的点。背驰一定发生在第二个及之后的中枢之后。 |
| **一卖** | 上涨趋势（≥2个同级别中枢）出现趋势顶背驰的点。 |
| **铁律** | 盘整背驰绝对不能产生第一类买卖点（27课明确定义）。 |

### 二、第二类买卖点

**17课原文：**

17课本身没有明确定义第二类买卖点。

**21课（买卖点分析的完备性）原文：**

> "第二类买点是和第一类买点紧密相连的，因为出现第一类买点后，必然只会出现盘整与上涨的走势类型，而**第一买点出现后的第二段次级别走势低点就构成第二类买点**，根据走势必完美的原则，其后必然有第三段向上的次级别走势出现，因此该买点也是绝对安全的。"
>
> "**第二类买点，不必然出现在中枢的上或下，可以在任何位置出现**，中枢下出现的，其后的力度就值得怀疑了，出现扩张性中枢的可能性极大，在中枢中出现的，出现中枢扩张与新生的机会对半，在中枢上出现，中枢新生的机会就很大了。但无论哪种情况，赢利是必然的。"
>
> "只有第二类买点与第三类买点是可能产生重合的，这种情况就是：第一类买点出现后，一个次级别的走势凌厉地直接上破前面下跌的最后一个中枢，然后在其上产生一个次级别的回抽不触及该中枢，这时候，就会出现第二类买点与第三类买点重合的情况"

**归纳——第二类买卖点的严格定义：**

| | 定义 |
|--|------|
| **二买** | 一买出现后，**第二段次级别走势的低点**。因为一买后只可能出现盘整或上涨，而第一段次级别走势向上（离开段），第二段次级别走势向下（回抽段），其低点就是二买。 |
| **二卖** | 一卖出现后，第二段次级别走势的高点。 |
| **约束** | 二买不必然不破一买低点——21课原文只说"第二段次级别走势低点就是二买"，**并未加"不低于一买价格"的条件**。理论上二买可以低于一买（即一买后上涨一段再跌创新低，这在更大级别视角下意味着一买可能判断错误，或者需要更高级别重新确认）。 |
| **位置** | 二买可出现在中枢上、中、下任何位置。 |

**⚠️ 关键歧义点：**

21课原文说的是"第一买点出现后的第二段次级别走势低点就构成第二类买点"，**并未附加"不低于一买价格"的条件**。但20课有相关描述——

**20课原文（缠师回复）：**

> "学好一招就可以了，例如，你把今天说的第三类买点搞清楚，在牛市中足以战胜95%的人。"

20课正文主要讲中枢扩张和三买定义，没有具体讨论二买的约束条件。关于二买"不低于一买"这个常见说法，实际上是很多缠论解读者的推导，而非原文的严格表述。

**但29课有一个相关论述：**

> "第一类买点是绝对安全的，即使是这样一种最低级别的反弹，也有足够的空间让买入获利"

这说明一买是绝对安全的，而二买也是绝对安全的（21课明确说"因此该买点也是绝对安全的"）。如果二买低于一买，那二买就是更好的买入点（更低的价格），但一买仍然是有效的。所以**原文的二买定义就是"一买后第二段次级别走势低点"，没有额外约束**。

然而，在实践中，很多缠论实现会加上"二买不低于一买价格"的约束，因为如果二买低于一买，意味着之前的趋势可能还在延续，一买可能是误判。这是一个实现层面的选择。

### 三、第三类买卖点

**17课原文：**

> 由定理一，可以得到第三类买卖点定理：**一个次级别走势类型向上离开缠中说禅走势中枢，然后以一个次级别走势类型回试，其低点不跌破ZG，则构成第三类买点；一个次级别走势类型向下离开缠中说禅走势中枢，然后以一个次级别走势类型回抽，其高点不升破ZD，则构成第三类卖点**。

**20课原文：**

> "缠中说禅走势级别延续定理一：在更大级别缠中说禅走势中枢产生前，该级别走势类型将延续。"
>
> 由此推导出第三类买卖点："一个次级别走势类型向上离开缠中说禅走势中枢，然后以一个次级别走势类型回试，其低点不跌破ZG，则构成第三类买点"

**21课原文：**

> "对于中枢下形成的买点，但如果该中枢是在上涨之中的，在中枢之下并不能必然形成买点，中枢下的买点，只可能存在于下跌与盘整的走势类型中。换言之，一个上涨趋势确定后，不可能再有第一类与第二类买点，只可能有第三类买点。"
>
> "而中枢扩张或新生，在中枢之上都会存在买点，这类买点，就是第三类买点。也就是说，**第三类买点是中枢扩张或新生产生的**。中枢扩张导致一个更大级别的中枢，而中枢新生，就形成一个上涨的趋势，这就是第三类买点后必然出现的两种情况。"

**20课评论区（缠师回复）：**

> 关于工商银行第三类买点的详细解释：
> "搞清楚，是看哪个级别的。对日线中枢来说，次级别是30分钟，而次级别的完成，需要再次级别，也就是5分钟图上呈现3段走势类型。这在工行14那天有着完美的表现。"

**归纳——第三类买卖点的严格定义：**

| | 定义 |
|--|------|
| **三买** | 一个次级别走势类型向上离开缠中说禅走势中枢，然后以一个次级别走势类型回试，其**低点不跌破ZG**（中枢上沿） |
| **三卖** | 一个次级别走势类型向下离开缠中说禅走势中枢，然后以一个次级别走势类型回抽，其**高点不升破ZD**（中枢下沿） |
| **前提** | 离开段必须**实际脱离中枢**：三买要求离开段高点>ZG，三卖要求离开段低点<ZD |
| **意义** | 三买意味着中枢扩张或新生，29课说"该趋势最后一个中枢的级别扩展"就是最弱的转折情况 |
| **与二三买重合** | 21课明确：一买后次级别走势直接上破下跌最后中枢，其上回抽不触及该中枢，二三买重合 |

**⚠️ 关于ZG/ZD的定义——原文精确表述：**

17课中枢定义：
> "某级别走势类型中，被至少三个连续次级别走势类型所重叠的部分，称为缠中说禅走势中枢。"

中枢区间[ZD, ZG]就是三个次级别走势重叠的区域。
- ZG = 中枢上沿 = 重叠区域的最高点
- ZD = 中枢下沿 = 重叠区域的最低点

三买：回抽低点 > ZG（中枢上沿之上）
三卖：回抽高点 < ZD（中枢下沿之下）

## 第二部分：当前代码实现逻辑

### 一、第一类买卖点代码（find_buy1_sell1，L128-184）

```
逻辑流程：
1. 遍历所有趋势背驰点 trend_bds
2. 对每个背驰点，找 zs.end_index <= bd.index 的中枢
3. 要求 related_indices.len() >= 2（至少2个中枢）
4. 将中枢按趋势方向分组（group_zs_indices_by_trend）
5. 对每组：group.len() >= 2（趋势至少含2个递进中枢）
6. 判断趋势方向（classify_trend_direction_from_indices）：
   - 上涨趋势：后ZD > 前ZG → "up"
   - 下跌趋势：后ZG < 前ZD → "down"
7. 方向匹配背驰方向 → 一买/一卖
```

**代码对齐审查：**

| 原文定义 | 代码实现 | 对齐？ |
|----------|----------|--------|
| 一买：下跌趋势底背驰 | 下跌趋势 + direction=="down" → 1buy | ✅ |
| 一卖：上涨趋势顶背驰 | 上涨趋势 + direction=="up" → 1sell | ✅ |
| 趋势≥2个同级别中枢 | group.len() >= 2 | ✅ |
| 盘整背驰不能产生一类 | 只取 trend 背驰（L83-86过滤panzheng） | ✅ |
| 铁律：单中枢不产生一类 | related_indices.len() >= 2 且 group.len() >= 2 | ✅ |

**⚠️ 潜在问题：**

1. **中枢分组可能把跨趋势的中枢错误归为同组**：如果存在先上涨后下跌的走势，两个上涨中枢和两个下跌中枢混在一起，`group_zs_indices_by_trend` 能正确分组吗？看代码逻辑——它是按连续同方向递进分组的，方向改变则新开一组，这应该是正确的。

2. **只看 bd.index 之前的中枢**：`zs.end_index <= bd.index` 只取背驰点之前结束的中枢。但如果中枢在背驰点附近正在延伸（end_index 略晚于 bd.index），可能漏掉。不过这是合理的——只有已经完成的中枢才能参与趋势判断。

### 二、第二类买卖点代码（find_buy2_sell2，L198-282）

```
find_buy2_after_buy1:
1. 从一买位置开始，找后续段
2. 遍历段，找第一个向上段（离开段），标记 found_up = true
3. 找到离开段后，看下一个向下段（回抽段）
4. 如果 pullback_low >= buy1.price → 二买
5. 否则 return None（一买可能无效）
```

```
find_sell2_after_sell1：
1. 从一卖位置开始，找后续段
2. 找第一个向下段（离开段），标记 found_down = true  
3. 找到离开段后，看下一个向上段（回抽段）
4. 如果 pullback_high <= sell1.price → 二卖
5. 否则 return None（一卖可能无效）
```

**代码对齐审查：**

| 原文定义 | 代码实现 | 对齐？ |
|----------|----------|--------|
| 二买 = 一买后第二段次级别走势低点 | 找到第一个向上段（离开）+ 第二个向下段（回抽），取其低点 | ⚠️ 有偏差 |
| 二卖 = 一卖后第二段次级别走势高点 | 找到第一个向下段（离开）+ 第二个向上段（回抽），取其高点 | ⚠️ 有偏差 |
| 不破一买价格 | pullback_low >= buy1.price | ❌ 原文无此约束 |
| 不破一卖价格 | pullback_high <= sell1.price | ❌ 原文无此约束 |

**⚠️ 关键问题1：二买"不破一买"是否原文要求？**

21课原文："第一买点出现后的第二段次级别走势低点就构成第二类买点"。原文**没有**附加"不低于一买价格"的条件。

但代码要求 `pullback_low >= buy1.price`，这实际上比原文更严格。

**支持加约束的理由**：如果二买低于一买，意味着一买后的走势创新低了，这在实践中说明一买可能判断错误（趋势还在延续）。很多缠论实操都采用"不破一买"作为二买的必要条件。

**反对加约束的理由**：21课原文没有这个约束。理论上二买低于一买（即一买后上涨又跌出新低），在更大级别上可能是重新组成趋势的一买，但在当前级别，只要一买后的第二段低点出现，就是二买。

**结论**：这个约束是合理的实践选择，但与原文定义不完全对齐。**建议保留此约束，但在文档中注明这是比原文更严格的实现**。

**⚠️ 关键问题2：二买是"第二段"还是"第一个回调段"？**

21课原文："第一买点出现后的第二段次级别走势低点就构成第二类买点"。

这里"第二段"是指一买之后的第二段走势。假设一买是某个向下段的终点：
- 第一段 = 一买后的向上段（离开段）
- 第二段 = 一买后的向下段（回抽段）
- 第二段的低点 = 二买

代码逻辑：找一买后的第一个向上段 + 下一个向下段 → 这就是"第一段和第二段"。✅ 对齐。

但有一个微妙的问题：**一买位置本身可能不是一个段的起点**。代码用 `s.start_idx >= buy1.index` 来过滤段，如果一买的index恰好是某个向上段的start_idx，那第一个过滤到的段就是离开段（正确）。如果一买的index在某个段的中间呢？

实际上，代码的 `buy1.index` 来自背驰点的index，而背驰点的index来自段的 `end_index`（趋势背驰点的index就是背驰段的end_index）。所以一买index=背驰段的end_index，下一个段的start_index应该等于这个end_index。所以 `start_idx >= buy1.index` 是正确的。

**⚠️ 关键问题3：如果一买后第一个段不是向上段怎么办？**

当一买在某个向下段的终点时，下一个段应该是向上段。但代码用 `found_up` 标记来跳过第一个向上段然后再找向下段。如果一买后第一个段恰好是向下段（意味着一买判断位置有问题），代码会跳过所有非向上段，直到找到第一个向上段。这在大多数情况下是正确的。

但21课说的更精确——"第二段次级别走势"，不论方向。即一买后，第一段走势（无论方向）之后的第二段走势的低点。代码假设第一段一定向上，如果走势符合缠论定义（一买后转化为上涨或盘整），第一段确实应该向上。所以这个假设在实践中是合理的。

### 三、第三类买卖点代码（find_buy3_sell3，L299-362）

```
逻辑流程：
1. 遍历所有中枢
2. 对每个中枢，找 end_index 之后的所有段
3. 扫描所有相邻段对：
   - 三买：leave_seg方向=up + back_seg方向=down
     + leave_high > zs.zg（离开段脱离中枢上沿）
     + back_low > zs.zg（回抽不破中枢上沿）
   - 三卖：leave_seg方向=down + back_seg方向=up
     + leave_low < zs.zd（离开段脱离中枢下沿）
     + back_high < zs.zd（回抽不破中枢下沿）
4. 每个中枢可能产生多个三买/三卖
```

**代码对齐审查：**

| 原文定义 | 代码实现 | 对齐？ |
|----------|----------|--------|
| 三买：次级别离开中枢向上 + 回抽低点不跌破ZG | leave_seg.up + back_seg.down + back_low > zg | ✅ |
| 三卖：次级别离开中枢向下 + 回抽高点不升破ZD | leave_seg.down + back_seg.up + back_high < zd | ✅ |
| 离开段必须实际脱离中枢 | leave_high > zg / leave_low < zd | ✅ |
| 离开和回抽是次级别走势类型 | 用笔/线段作为次级别单位 | ✅ |
| ZG/ZD含义：ZG=中枢上沿，ZD=中枢下沿 | 代码直接用 zs.zg / zs.zd | ⚠️ 需确认 |

**⚠️ 关键问题1：ZG/ZD精确值——严格不等还是非严格不等？**

17课原文："其低点不跌破ZG"。

"不跌破ZG"在严格意义上是 > ZG，还是 >= ZG？

原文20课评论区回复工行例子中讨论了这个问题。代码用的是 `back_low > zs.zg`（严格大于）。

如果回抽低点恰好等于ZG，算不算三买？严格来说，"不跌破"包含"恰好触及"，即回抽低点 >= ZG 也算不跌破。但"等于ZG"意味着回抽恰好触及了中枢的上边界，这在缠论原文中需要更细致的分析——原文20课定义中枢时说的是"重叠部分"，等于边界是否算重叠？

**我的判断**：`>` 比 `>=` 更安全，因为如果回抽触及ZG，从保守角度看已经进入中枢区间。但原文"不跌破ZG"更接近 `>=`。

目前代码用 `>` 是合理的保守实现。**建议暂不修改，但在文档中注明**。

**⚠️ 关键问题2：三买与离开段起起始位置的关系**

原文说的是"次级别走势类型向上离开中枢"，这意味着离开段应该是**从中枢内部出发**的。代码只要求离开段在 中枢结束（start_idx >= zs.end_index）之后出现，但并没有验证离开段的起点是否在中枢内部。

理论上，中枢之后的段可能并不一定是从中枢内部出发的。但目前实现中，中枢的end_index就是中枢的结束位置，之后的段自然是从中枢附近出发的。这是合理的近似。

**⚠️ 关键问题3：同一中枢可能产生重复三买/三卖**

代码扫描中枢后所有相邻段对，只要满足条件就产生三买/三卖。这意味着同一中枢可能产生多个三买。这在原文中是合理的——中枢后多次离开都是可能的，每次合法的离开+回抽都可以构成三买。

但代码中没有对重复的三买进行去重（只对index+bs_type相同的去重，L104）。如果同一个位置被两个不同的段对都判为三买，可能会出现重复。但由于dedup，这个问题应该不严重。

**⚠️ 关键问题4：二三买重合的情况**

21课明确说："但第一类买点出现后，一个次级别的走势凌厉地直接上破前面下跌的最后一个中枢，然后在其上产生一个次级别的回抽不触及该中枢，这时候，就会出现第二类买点与第三类买点重合的情况"

当前代码中，二买和三买的检测是独立的（find_buy2_sell2 和 find_buy3_sell3 分别执行），如果某个位置同时满足二买和三买的条件，会在 results 中出现两个相同index但不同bs_type的点。代码用 `dedup_by(|a, b| a.index == b.index && a.bs_type == b.bs_type)` 去重，但不会将同一位置的"2buy"和"3buy"合并为"2+3buy"之类的标记。

这是一个功能缺失——**二三买重合是一个重要的信号（21课说"一旦出现这种情况，一个大级别的上涨往往就会出现"）**，应该标记出来。
