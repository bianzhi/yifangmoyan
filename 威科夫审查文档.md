### reason 格式

每条供需线的 `reason` 字段包含：
- **趋势线型**：`供给线(趋势线): 连接swing点[#N@价格→#N@价格], 斜率=X.XXXX(上升/下降/水平), 角色说明`
- **区间边界型**：`供给线(区间上沿): 交易区间[#N~#N]上沿=X.XX, 由AR高点确立的供给边界`
- **冰线型**：`冰线(ICE): 交易区间中位线=X.XX, 区间宽=X.XX(X.X%), 跌破冰线→供给压倒需求`

## 七、本次修改总结

### Rust 端

1. **WyckoffEvent 增加 `reason: String` 字段**：每个事件都携带详细判断依据
2. **SupplyDemandLine 增加 `reason: String` 字段**：每条供需线都说明构造方式
3. **JOC 改写**：
   - 函数签名增加 `events` 参数，查找关联的AR事件
   - `reason` 中包含小溪宽度（绝对值+百分比）
   - `reason` 中说明小溪由哪个AR高点到哪个低点构成
4. **所有事件 reason 格式统一**：`事件类型(中文名): 具体数值判断, 原文定位（Wyckoff Phase X）`

### 前端

1. **接口更新**：`WyckoffEvent` 和 `SupplyDemandLine` 增加 `reason: string`
2. **光标行显示**：
   - 威科夫事件：事件类型 + reason（截断+hover展开）
   - 供需线：类型简写 + 价格 + reason
   - 交易区间：下沿~上沿 + 冰线
   - 阶段：Phase + 子阶段
   - 努力与结果：协调/背离 + 主导方
3. **tooltip 显示**：
   - 威科夫事件：类型 + description + 完整reason
   - 供需线：类型 + 价格区间 + 斜率 + 完整reason（构造说明）
4. **新增 `WYCKOFF_EVENT_DESC` 常量**：14个事件的原始定义

### JOC 小溪宽度说明

威科夫原著中，Robert Evans 将 SOS 比喻为"跳过小溪"(Jump Across Creek)：
- **小溪** = 交易区间上沿（AR高点确立的供给线/阻力区）
- **小溪宽度** = 阻力线价格 - 交易区间下沿（SC低点）= 交易区间振幅
- **JOC** = 价格放量跳过这条"小溪"，需求强劲越过供给边界
- **LPS** = 跳过后回踩小溪边缘（Back Up to Creek），是最佳入场点

当前代码中 `creek_width = resistance - recent_low`，计算方式与原文一致。
